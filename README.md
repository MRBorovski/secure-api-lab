# Secure API Lab

## Опис
Цей проєкт реалізує REST API на Node.js з використанням Express. API дозволяє працювати з ресурсами **documents** та **employees**, забезпечуючи аутентифікацію та авторизацію за ролями. Користувачі з роллю `user` можуть отримувати та створювати документи, а користувачі з роллю `admin` мають доступ до всіх документів та списку співробітників.

## Middleware

LoggingMiddleware — логування всіх запитів (метод, URL, timestamp)
AuthMiddleware — аутентифікація через заголовки X-Login та X-Password
AdminOnlyMiddleware — доступ лише користувачам з роллю admin

## API Ендпоінти

| Method | URL            | Headers             | Body                                         | Status Codes  | Опис                                                             |
| ------ | -------------- | ------------------- | -------------------------------------------- | ------------- | ---------------------------------------------------------------- |
| GET    | /documents     | X-Login, X-Password | -                                            | 200, 401      | Повертає список всіх документів (для авторизованих користувачів) |
| POST   | /documents     | X-Login, X-Password | `{ "title": "string", "content": "string" }` | 201, 400, 401 | Створює новий документ                                           |
| DELETE | /documents/:id | X-Login, X-Password | -                                            | 204, 401, 404 | Видаляє документ за ID                                           |
| GET    | /employees     | X-Login, X-Password | -                                            | 200, 401, 403 | Повертає список співробітників (тільки admin)                    |

## HTTP статуси
200 OK — успішне виконання запиту
201 Created — ресурс успішно створено
204 No Content — ресурс успішно видалено
400 Bad Request — некоректні дані запиту
401 Unauthorized — невірні креденшали
403 Forbidden — доступ заборонено (роль не admin)
404 Not Found — ресурс не знайдено
